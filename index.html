WEEK 3 Practice

Understanding CSS Pre-Processor: SASS/SCSS

SASS stands for Syntactically Awesome StyleSheet. It's a CSS preprocessor that is sometimes referred to a CSS extension. It allows developers to use features that does not exist in normal CSS. Since the browser does not read SASS directly, it has to be compiled with CSS compiler.
There are two different methods/file extension when writing SASS: the .scss extension and .sass extension.

Features of SASS

Variables

This is a way of storing custom properties to a particular keyword using the dollar sign "$". e.g $primary-color: blue. This way, it is easier to read, write and remember than CSS custom properties.

Nesting

This is an easier way of styling child selectors of HTML elements. It makes it easier to target and style each child selector within an element.

Modules

SASS can be divided into seperate files and bring them into a main SASS file with Modules. SASS makes it easier with compiler. Different components like buttons, navbars, etc can be created in different SASS files and compiled together in the main files

Mixin & Functions

This allows developers to create blocks of CSS codes and functions that can be reused later.

Inheritance

Different properties values can be extended to other properties or elements.
